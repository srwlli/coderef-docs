Know these by name

The canonical registries

The registries worth knowing by name — all under TRACKING/, the view-typed ones regenerated from the DB, never edited. One of them, agent-domains.json, is the source of truth for "who am I" across the whole multi-agent system.

The registries

All live under TRACKING/; the view-typed ones project from unified-registry.json (or a named DB table) and are regenerated, not edited.

The identity registry

One registry deserves a special note: agent-domains.json is the map the cwd resolver uses to decide which domain an agent is, by matching the agent's working directory against each domain's working_directory_glob (deepest-match-wins). Identity comes from where you are on disk right now — not from memory, not from a hardcoded flag. That's why this file is load-bearing: it's the source of truth for "who am I" across the whole multi-agent system.

Canonical sources

  • TRACKING/registries.json — the registry of registries.
  • TRACKING/agent-domains.json — the identity map.
