The engine

What is CODEREF-CORE

CODEREF-CORE is the standalone code-intelligence engine of the ecosystem — a scanner + retrieval core delivered as a CLI. It reads a codebase, produces the .coderef/ artifact set, and builds semantic search over the same code. Everything else consumes its artifacts rather than re-scanning.

The engine in one view

CODEREF-CORE is the standalone code-intelligence engine of the ecosystem: a scanner + retrieval core delivered as a CLI (@coderef/core, run via npx @coderef/core <command> or the locked local checkout at CODEREF/CODEREF-CORE).

It does two jobs:

  • Scan — read a codebase and produce the .coderef/ artifact set: an element-level index, a dependency graph, and report files (patterns, coverage, drift, complexity).
  • Retrieve — build and serve a semantic search index over the same code, so agents can ask natural-language questions of a codebase.

The key consequence: everything else in the ecosystem that wants to know something about a codebase — planners, validators, doc generators — consumes CODEREF-CORE's artifacts rather than re-scanning. One engine scans; everyone reads.

The .coderef/ artifact set

A scanned project carries a .coderef/ directory holding:

  • The element-level index — what is defined where, down to individual elements.
  • The dependency graph — what depends on what.
  • Report files — patterns, coverage, drift, and complexity reads over the code.
  • The semantic search index — vector embeddings built by the rag family (see Semantic search).

Thin skills over one CLI

Skills in the CORE category are deliberately thin: each one wraps exactly one CLI verb, so agents invoke the engine through the same governed surface as every other skill. The rule of thumb: the CLI is the engine; the CORE skills are the governed way agents drive it.
