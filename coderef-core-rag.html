Semantic search (the rag family)

The rag skills build a vector index over a project's code and answer natural-language questions against it — rag-index to build, rag-search to query, rag-status to inspect, and coderef-rag-server as the REST surface other tools call.

Build the index

/rag-index builds the semantic search index — vector embeddings — over a project's code. This is the prerequisite for every query that follows; the index lives alongside the other .coderef/ artifacts.

Query it

/rag-search is natural-language semantic code search against that index. The canonical example: "where do we validate session tokens?" — a question about meaning, not a grep for a string. The engine returns the code locations whose semantics match.

Inspect it

/rag-status is the read-only inspection surface: index metadata such as freshness and size. Use it to answer "is this index current enough to trust?" before querying.

Serve it

/coderef-rag-server manages the lifecycle of the RAG HTTP server — start, query, index, status, reset. This is the REST surface other tools call at query time, so search is available to anything that can make an HTTP request, not just an agent at a terminal.
