The fleet's heartbeat

The daily session

A multi-agent day runs inside a daily session — a dated directory every domain joins, where work moves by dispatch and idle agents stay responsive on a wake-poll loop. This is the skill-side of how the loop runs.

The session is the day

A multi-agent day runs inside a daily session — a dated directory under LOGS/SESSIONS/daily-agent-session-{date}/. The session is the heartbeat that ties the fleet together: it is created once at the start of the day, every domain joins it, work moves between domains through it, and it is swept closed at day's end.

This section covers the skill-side — the mechanics, run by the SESSION skills — and, composed from ASSISTANT's own words, the coordination side: How the day is coordinated.

The three mechanics

Three moving parts make the session work, each its own page:

  • The lifecycle — create the session, join it, close it. How a domain checks in and how the day ends.
  • The dispatch flow — how work moves between domains: file a dispatch, accept it, complete it, read the result.
  • The wake-poll loop — how an idle agent stays responsive: sleep, poll the inbox, re-arm. A loop that forgets to re-arm dies silently.
