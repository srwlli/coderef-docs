The coordination side

How the day is coordinated

ASSISTANT opens the session, every agent checks in and arms a wake-poll loop, work flows as dispatches (files in inboxes, polled — no server), blockers route back as andon pulls, and the day ends with a liveness-aware close. Coordination is a protocol made of files and a rhythm, not a runtime.

The session is the heartbeat

A daily session is how the fleet works as one thing for a day. It is not a meeting and not a server — it's a directory on disk (LOGS/SESSIONS/daily-agent-session-{date}/) that every active agent checks into, plus the coordination ASSISTANT runs on top of it. When you hear "the loop," this is it: session-open, agents check in, work routes, agents stay responsive, session-close. The session is what turns a pile of independent agents into a coordinated fleet for the span of a day.

This page is the coordination half of the section — how the day is driven. The lifecycle, dispatch flow, and wake-poll pages cover the skill-side mechanics.

How ASSISTANT runs the day

ASSISTANT opens the session (/create-daily-session) — it seeds a directory for each domain that worked the prior day, writes the session record, and registers it in the active-sessions manifest so the resolver can find it. From that moment the day is "live."

Agents check in (/join-daily-session) — each agent registers its presence and arms its wake-poll loop. After this, an agent is in the session: visible, reachable by dispatch, and on the heartbeat.

ASSISTANT coordinates, it does not micromanage. Its job for the day is to route work to the right domain, absorb blockers so no one is stuck, integrate finished work back to the trunk, and keep the standards. It does not do the domains' work — each domain is sovereign over its own area; ASSISTANT moves work between them.

How work routes

Work moves through the fleet as dispatches — small JSON files dropped into a domain's inbox: from-domain, to-domain, a task, an objective, and — when it's real build work — the skills the receiver must log to complete it. The receiver picks it up on its next poll, accepts it, does the work, and marks it complete.

This is the whole routing model: no central scheduler, no queue server — just files in inboxes that agents poll. ASSISTANT routes a dispatch to the domain that owns the work; when a domain finishes something that needs landing, it dispatches a ready-to-integrate notice back to ASSISTANT (the integrator). The fleet coordinates by passing files, not by a runtime.

Two dispatch shapes are worth knowing:

  • Work-delegation — carries required skills and is strict-enforced: the receiver must run the full lifecycle.
  • Instruction-pass / recommendation — no strict lifecycle: a "do this" or a "flagging for your call." Tagging a recommendation as strict is a known foot-gun the system has since fixed.

How blockers route back

A core rule: agent terminals are execution surfaces, not decision surfaces. When an agent hits something it can't decide — a blocker, an ambiguous scope, a cross-domain call — it does NOT ask the operator directly and does NOT guess. It routes the decision back to ASSISTANT as a blocked dispatch: the andon pull, like an assembly-line worker pulling the cord.

ASSISTANT then either decides autonomously (most things are standards calls it can make) or, for the small operator-reserved set — schema-breaking changes, irreversible actions, explicit asks — surfaces it to the operator. The operator decides; ASSISTANT routes the resolution back down. Decisions flow to the one place equipped to make them, and agents keep executing instead of stalling on questions that aren't theirs to answer.

How the loop ends

At day's end ASSISTANT closes the session (/close-daily-session). The close is liveness-aware — it refuses to close a session that's still genuinely active (recent agent activity within the window), so the day doesn't get pulled out from under working agents. A truly stale session closes cleanly; a live one is protected. When it closes, the session record is timestamped done and the day's heartbeat stops.
