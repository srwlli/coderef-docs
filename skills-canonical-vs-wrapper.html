One source of truth, many surfaces

Canonical vs wrapper

A skill's real logic lives in exactly one place — the canonical package in the ASSISTANT repo. What's installed into each CLI surface is a thin wrapper that points back at it. Validators enforce the split.

The split

The skill's real logic lives in one place — the canonical package under SKILLS/ in the ASSISTANT repo. What's installed into each CLI surface (~/.claude/skills/, ~/.codex/skills/, ~/.gemini/antigravity-cli/skills/) is a thin wrapper: frontmatter plus a Read: pointer to the canonical SKILL.md and the instruction "do not define a second workflow here."

Wrappers are delivery surfaces, never workflow owners. Three CLI tools can invoke the same skill, and all three are reading the same canonical contract — fix it once, every surface gets the fix.

The wrapper contract

A wrapper carries no workflow of its own. Its whole job is routing: name the skill, point at the canonical file, get out of the way. The moment a wrapper grows steps, the system has two versions of the truth — which is exactly the failure the model exists to prevent.

The runner-short-circuit profile

One refinement exists for hot-path loop skills: the runner-short-circuit wrapper profile tells the agent to run the skill's runner directly on a normal tick instead of re-reading a large canonical file every few minutes. Same single source of truth — far cheaper context. The wake-poll inbox check is the canonical example: it fires every few minutes all session long, and the wrapper inlines just enough to act on the runner's output without loading the full contract each tick.

Enforcement

Validators enforce all of this mechanically:

  • Package shape — the directory anatomy is checked, not trusted.
  • Three-surface wrapper parity — the installed wrappers across the CLI surfaces must agree.
  • Signpost discipline — descriptions that drift into changelog history are refused.
  • The version-bump gate — a pre-commit hook refuses skill-code changes that don't bump the version.

Canonical sources

  • SKILLS/WORKFLOW/new-coderef-skill/SKILL.md — installs and maintains the wrappers; its references/ hold the packaging and wrapper standards.
  • SKILLS/STANDARDS/SKILL-CONTRACTS.md — the enforcement-contract taxonomy.
