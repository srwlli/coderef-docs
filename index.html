The front door

What is CodeRef

CodeRef is a multi-agent software ecosystem — a set of AI agents, each owning a domain, that coordinate through files and standards rather than a central app. This site is the front door: arrive here, understand the system, branch into any part.

A coordination protocol

CodeRef is a multi-agent software ecosystem — a set of AI agents, each owning a domain, that coordinate through files and standards rather than a central app. There is no server orchestrating them and no database holding their shared state: coordination happens through plain files on disk (dispatches, session logs, a tracking registry) that every agent reads and writes.

An agent is told how to work by the standards, not what the answer is by a knowledge base — so the system scales by adding capabilities, not by memorizing codebases.

This docs site is the readable surface over that protocol. It is a guide: where you arrive to get oriented, then drill into the part you need. The canonical truth lives in the repository — the SKILL.md files, the STANDARDS/, the registries — and this site points you at it.

The load-bearing ideas

Read these four and you understand the rest:

  • File-based coordination, not a runtime. Agents talk by writing files — a dispatch is a JSON file dropped in another agent's inbox; a session is a directory; the registry is a SQLite DB exported to JSON views. Nothing depends on a live server being up.
  • Standards over knowledge. The system doesn't try to know every codebase. It enforces how work is done — where you commit, how a work order closes, how a bug gets tracked — and lets each domain own the knowledge of its own area. Correctness comes from mechanical gates (pre-commit hooks, validators), not from any agent remembering rules.
  • Skills are capability. Everything an agent can do is a skill — a documented, versioned unit (168 of them across 17 categories). A skill is a map-and-guide, not a script: it tells the agent what to do and points at the canonical source for depth.
  • The registry is the source of truth; the views are projections. State lives in a registry DB; the JSON files everyone reads are generated from it. When the view and the DB disagree, the DB wins — and a lot of the system's discipline is about never hand-editing a generated view.

If you take one thing from this page: CodeRef is a coordination protocol made of files and standards, run by agents who each own a domain.
