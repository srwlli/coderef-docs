Create, join, close

The session lifecycle

A daily session is created once, joined independently by each domain, and swept closed at day's end. One subdirectory per domain means no file locking is needed.

Create the session

A daily session is a dated directory under LOGS/SESSIONS/daily-agent-session-{date}/, created once by /create-daily-session. That one call sets up the day's shared space; everything else hangs off it.

Join the session

Each agent domain then joins independently with /join-daily-session. The join runner:

  • validates the session exists and is active,
  • writes the domain's checkin.json and status.json — no file locking needed, because each domain gets its own subdirectory,
  • runs the mandatory cultural-alignment gate, and
  • seeds the wake-poll loop.

Because every domain writes into its own subdirectory, many agents can join the same session concurrently without colliding.

Close the session

At day's end, /close-daily-session sweeps the session closed. It runs a git-doctor backstop that surfaces any stranded work — a branch committed but never merged, a worktree left dirty — so nothing quietly falls through the cracks as the day ends.
