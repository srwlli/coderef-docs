Visualizations

Interactive views of the ecosystem. First up — the typed skill-dependency graph, which renders depends_on and supersedes as visually distinct edges so the graph never flattens to an untyped link map.

The git subgraph

Below is the git category's typed dependency subgraph — five skills and the real depends_on relations between them, drawn live from the skill manifests. This is the first embed of the reusable typed-skill-graph component; more categories (and eventually the full 170-skill map) follow.

Reading the graph

Edges are typed, and the type is never flattened away:

  • Solid arrow &mdash; depends_on (A relies on B).
  • Dashed arrow &mdash; supersedes (A replaces a retired B).

The legend always shows both edge styles, even though the git category happens to be all depends_on — that is the no-flatten guarantee made visible. The distinction is carried by stroke style (solid vs dashed), not color alone, so it survives greyscale and color-blindness. Hover any node or edge for its full label and relation.

How this is built

The graph is a reusable component owned by SURFACES-HTML; this docs site consumes it as an embeddable leaf (one render(mountEl, {nodes, edges}) call, zero imports, no runtime framework). The site vendors the component as-is and bridges its design tokens to the docs theme — it does not fork the component. The node and edge data come from TRACKING/skills.json plus the per-skill skill.json manifests; reads/writes are omitted by design because they point at file globs, not skills.
