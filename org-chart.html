How the fleet is arranged

The org chart

The ecosystem has 63 registered domains (35 active), grouped by project. The shape is fractal — ASSISTANT coordinates the ecosystem the way a project coordinator drives its own project, one level down.

The shape is fractal

The ecosystem has 63 registered domains (35 active), grouped by project. The shape is fractal: ASSISTANT coordinates the whole ecosystem the same way PRIMARY-SOURCES coordinates its own project — one level down. A "lead" domain coordinates a small team under it; the pattern repeats.

At the top sits the operator, who drives through ASSISTANT. Below ASSISTANT, the fleet falls into three groups: the ecosystem tooling, the coordinated projects, and the standalone projects.

Ecosystem tooling

About 18 active domains sit directly under ASSISTANT (registry project_id = assistant). They are the machinery the whole ecosystem runs on:

The PRIMARY-SOURCES project

PRIMARY-SOURCES is a project coordinator — it drives the PS project the same way ASSISTANT drives the ecosystem, one level down. Under it, lead domains each coordinate a small team:

  • PS-APP (front-end lead) — leads PS-HTML, PS-REVIEWER
  • PS-ARCHIVE (lead) — leads SOURCE-MATERIAL, PS-RESEARCH
  • PS-PIPELINE (lead) — leads PRIMARY-SOURCES-API, PS-TRANSCRIBE
  • PS-DEBUG — the project's debug agent
  • STAFF — a docs/standards shelf, not a coordinator (see the note below)

Standalone projects

These stand on their own, outside the ASSISTANT tooling group and the PS project:

  • CODEREF-CORE — the CLI / core engine
  • ENTITY — the entity-platform project
  • ON-TASK — the operator focus / task-board layer
  • PROJECTS — the project registry root

A note on the registry

One trap worth calling out: STAFF looks like a tier in the registry's hosts_under field, but it is just a docs/standards folder where agent files live — not a boss. It is not a coordinator.

More generally, hosts_under records where an agent's files are filed, which isn't always the same as who coordinates it. The canonical command line is the project lead, not the folder.
